## 0.1.2 滚动空白修复

- 根因是扩展把自己写入卡片外层的固定高度再次作为下一轮测量结果；Pinterest 在滚动中复用或更新 Pin 后，旧高度可能只增不减，从而在卡片下方留下大块空白。
- 普通卡片改为直接测量其唯一 Pin 的真实高度，轮播等多 Pin 卡片才回退到内部内容包装节点，不再读取扩展控制的外层 `offsetHeight` 或 `scrollHeight`。
- 使用 `ResizeObserver` 监听实际测量节点，图片或文字造成内容高度变化时自动重新排版。
- 1600 × 1000、5 列状态下连续滚动并采样 80 次：外层占位与真实内容最大高度差 0px，异常高度 0 次，未就绪 0 次，坐标样式中断 0 次，未定位卡片 0 次，可见重叠 0 次；同列卡片最大间隔为预期的 16px。
- 另外检查 1920 × 1200、5 列深滚动状态，画面中没有卡片间异常空洞、重叠或闪烁。
- 回归测试增加“高度测量不能读取扩展控制的外层高度”和“内容尺寸变化会触发重排”两项检查。
- 截图：`artifacts/pinterest-1600-5-columns-no-gaps-scrolled-final.png`、`artifacts/pinterest-1920-5-columns-no-gaps-scrolled-final.png`。

## 0.1.1 固定列数闪烁修复

- 将卡片基础尺寸和最终坐标拆到两个独立样式层；重新测量时不再清空上一帧有效坐标。
- `MutationObserver` 只响应瀑布流容器和 Pin 节点变化，忽略弹窗、悬浮层等无关 DOM 更新。
- 新卡片在获得自定义坐标前保持隐藏，避免短暂使用 Pinterest 原坐标造成重叠。
- 1600 × 1000、3 列状态下进行 3 秒高频图片加载和无关 DOM 变化压力测试：189 帧中定位缺失 0 帧、未就绪 0 帧、坐标变化 0 次。
- 继续进行 5 秒滚动测试：101 个可见采样帧中定位缺失 0 帧、未就绪 0 帧、可见重叠 0 帧；新增卡片最多由 12 增至 23。
- 自动模式复测：容器、卡片、已定位标记及两个覆盖样式均完整清除。
- 截图：`artifacts/pinterest-1600-3-columns-no-flicker-final.png`。

## 发现并修复的问题

1. Pinterest 的卡片定位带有 `transform` 过渡，重排时会造成短暂横向抖动。扩展接管布局时现已关闭容器和卡片过渡。
2. 监听所有卡片 `style` 属性会被 Pinterest 的持续定位更新频繁触发，使最终排版反复中断。现改为只监听新增节点，并继续通过图片加载、窗口变化和路由变化触发重排。

## 已知限制

- 隔离测试 Chrome 没有 Pinterest 登录态，Pinterest 会显示登录遮罩；背景中的真实搜索瀑布流及其几何数据已经验证，但登录后的首页、画板和个人主页需要在用户已登录的日常 Chrome 中继续做一轮使用验收。
- Pinterest 属于持续更新的第三方页面，DOM 结构大改时可能需要更新容器识别逻辑。
