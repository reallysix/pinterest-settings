# Pinterest 图片列数控制扩展研发方案

## 1. 项目目标

为桌面版 Google Chrome 开发一个仅作用于 `pinterest.com` 的 Manifest V3 扩展。用户可以指定 Pinterest 瀑布流的列数，让图片在宽屏显示器上以更大的尺寸展示，适合数码自媒体拍摄时作为屏幕背景使用。

首版不下载图片、不采集页面内容、不接入服务器，所有设置仅保存在浏览器本地或 Chrome 同步存储中。

## 2. 首版范围

### 必须完成

- 扩展开关。
- 自动布局及 2、3、4、5、6 列切换。
- 记住用户最后一次选择。
- 刷新 Pinterest 页面后自动生效。
- 适配首页、搜索结果、画板和个人主页中的瀑布流。
- 适配无限滚动新增卡片。
- 适配窗口尺寸变化和 Pinterest 站内路由切换。
- 关闭扩展或选择“自动”时恢复 Pinterest 原始布局。

### 首版不做

- 图片下载、批量采集或高清原图解析。
- 用户账号、云端配置或数据统计。
- 自动滚动、广告过滤、沉浸模式及图片筛选。
- Firefox、Safari 和移动端适配。

## 3. 技术结论

Pinterest 当前使用 JavaScript 计算的绝对定位瀑布流，而不是普通 CSS Grid。单纯覆盖 `grid-template-columns` 无法可靠改变列数。

扩展需要识别实际卡片容器，改变卡片宽度，并使用“最短列优先”算法重新计算每张卡片的位置及容器高度。布局还需要在以下事件后重新执行：

- 新卡片通过无限滚动加入 DOM。
- 图片加载完成导致卡片高度变化。
- 浏览器窗口尺寸变化。
- Pinterest 站内跳转到新页面。
- 用户切换列数或启用状态。

## 4. 技术架构

### Manifest V3

- `manifest.json`：声明扩展入口、Pinterest 匹配范围和最小权限。
- `src/content/layout.js`：纯布局计算函数。
- `src/content/content.js`：容器识别、样式注入、DOM 监听及生命周期。
- `src/popup/`：列数选择和启用开关。
- `chrome.storage.sync`：保存启用状态和列数。

### 权限策略

仅申请：

- `storage`：保存用户设置。
- `https://*.pinterest.com/*` 页面访问范围：注入内容脚本并修改布局。

不申请 Cookies、下载、浏览历史、标签页或网络请求权限。

### 布局算法

1. 计算可用容器宽度。
2. 根据列数和间距计算卡片宽度。
3. 先应用卡片宽度，等待浏览器完成一次排版。
4. 获取卡片实际高度。
5. 每次将下一张卡片放入当前累计高度最小的列。
6. 更新卡片位移及瀑布流容器高度。

## 5. 稳定性设计

- 优先使用 Pinterest 的 `data-test-id`，避免依赖随机生成的 CSS 类名。
- 使用结构特征作为选择器降级方案。
- 使用 `MutationObserver` 监听新增卡片，且只监听节点变化，避免扩展自身样式更新造成循环。
- 对滚动、窗口变化和 DOM 更新进行防抖，避免频繁重排。
- 使用独立 `<style>` 元素覆盖 Pinterest 内联定位，减少 React 重渲染覆盖扩展结果的概率。
- 页面失去目标容器时清理旧标记，等待新页面内容出现后重新识别。

## 6. 测试与验收

### 自动化检查

- 不同列数下的卡片宽度计算。
- 最短列放置顺序和最终容器高度。
- 空卡片列表、单列及不同卡片高度。
- 设置读取、保存和非法值回退。

### 真实界面检查

- 分辨率：1366px、1920px 和 2560px 宽屏。
- 浏览器缩放：80%、100% 和 125%。
- 状态：自动、2 列、3 列、6 列、关闭扩展。
- 页面：首页、搜索、画板、个人主页。
- 生命周期：首次加载、刷新、连续滚动、站内跳转、调整窗口。

### 完成标准

- 卡片无明显重叠、横向溢出或异常大面积空白。
- 图片保持原始宽高关系，不被扩展拉伸。
- 新加载卡片自动进入指定列数布局。
- 弹窗控件可读、可点击，当前状态表达明确。
- 关闭扩展或选择自动后恢复原页面布局。

## 7. 研发阶段

| 阶段 | 内容 | 预计时间 |
| --- | --- | --- |
| 技术原型 | Pinterest 结构识别和 2–6 列重排 | 0.5–1 天 |
| 核心开发 | 布局算法、监听器和恢复机制 | 1–2 天 |
| 扩展界面 | 弹窗、设置持久化和状态反馈 | 0.5–1 天 |
| 稳定性适配 | 无限滚动、路由、窗口和图片加载 | 1–2 天 |
| 测试交付 | 自动化检查、真实截图和安装说明 | 1–2 天 |

个人自用 MVP 预计 3–5 个工作日；达到适合公开上架的稳定程度预计 5–8 个工作日，不包含 Chrome Web Store 审核等待时间。

## 8. 后续路线

1. 沉浸展示模式：隐藏导航栏、搜索框、文字和操作按钮。
2. 自定义图片间距、圆角和页面背景色。
3. 快捷键切换列数和沉浸模式。
4. 自动匀速滚动。
5. 横图、竖图和宽高比筛选。
6. 单图全屏轮播。

图片下载或批量采集功能需要另行评估版权、隐私、Pinterest 兼容性和 Chrome Web Store 审核风险。
